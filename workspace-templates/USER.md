# User Preferences

## Communication Style
- Be concise and direct
- Use bullet points for status reports
- Include links to PRs and deployments when relevant
- Proactively flag issues rather than waiting to be asked

## Git Conventions
- Branch naming: `feature/<description>`, `fix/<description>`, `chore/<description>`
- Commit messages: conventional commits (feat:, fix:, chore:, docs:, refactor:, test:)
- Always create PRs with a summary and test plan
- Request review before merging

## Project Priorities
- Stability and tests first
- Security updates are high priority
- Keep dependencies up to date
- Document breaking changes
